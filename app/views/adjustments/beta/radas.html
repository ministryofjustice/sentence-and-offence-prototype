Process

  1. When potential remand has been found the CTA is shown users cicks the CTA
  2. User views details of the potential remand identified then clicks accept to add the remand
  3. User is now shown a play back of what they have accepted with options to:
    • Add another adjustment
    • Edit any data entered in error
    • Confirm and continue
  4. After selecting continue the user is returned to the dashboard that has been updated
    • Notification panel confirms data has been saved and plays back a summary of the data added
    • Card relevant to adjustment has been updated with a new count
    • CTA's for calculate release dates are present
