Adjustments prototypes

Details of usability testing for the adjustment service you can find design Iterations here.

Round 1

What did we test?

  • Providing a system for user to enter adjustments data based on the adjustment type
  • Displaying results of remand days calculation based on data available in NOMIS.

What did we found out

  • Users requested a better way to illustrate relationships between cases
  • Users needed more information about the offences (dates and outcomes)
Research playback board

Round 2

What did we test?

  • Journey for adding ADA's
  • Journey for adding RADA's
  • Journey for adding UAL's
  • Soft testing of hub concept
  • Iteration of remand calculation tool
    • grouping by date
    • Add offence dates
    • Include offence out come

What we found

  • Users expected days to be pre populated
  • Users requested ability to add multiple adjudication numbers
  • Users requested ability to add consec and concurrent
  • Users identified the need to see inactive remand periods
  • Users found the hub more useful than radio buttons
