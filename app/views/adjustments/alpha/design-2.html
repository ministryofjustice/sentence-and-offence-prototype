Adding other remand types introducing hub conceptRound 2

What we did

Building on the prototype from the first round we created journeys for three more adjustment types ADA, RADA and UAL. These adjustments type had differing data needs

After entering the data users are then taken to a review screen where they can review the data they entered for each adjustment type, they are also presented with an option to add other adjustment or confirm the data and end the journey.

We also linked users to the policy documentation

We also tested a concept of a more interactive adjustment type selection page that gave users more options when dealing with adjustments

The journey allows them to select the adjustment type enter the dat then review the data add more adjustments or continue to confirmation

What we found

  • User found the concept more useful
  • User were able to complete journeys for entering data
  • They found the journey easier than using NOMIS

What user said

  • They would like the ability to add comments when entering adjustment data
  • They expected the number of days to auto populate once they had entered the to and from dates
  • They have further data entry needs for some adjustments
  • They found the journey easier than using NOMIS
