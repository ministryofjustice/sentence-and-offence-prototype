Reject remand

Process

  1. When potential remand has been found the CTA is shown users cicks the CTA
  2. User views details of the potential remand identified then clicks reject to reject the remand
  3. User is then taken to a page to confirm they want to reject the identified remand. This page details:
    • Why the data may be incorrect
    • Ways to fix the data
  4. After selecting continue the user is returned to the dashboard that has been updated:
    • Notification panel confirms data has been rejected
    • Add remand (manually) cta is now available for the user to add remand
  5. optional User selects add remand and is able to continue to add remand as per the add single adjustment journey
